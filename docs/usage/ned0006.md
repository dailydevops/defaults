# NED0006: Missing Repository URL Configuration

## Overview

**Severity**: Warning  
**Category**: Package Metadata  
**Applies To**: Packable Projects

## Description

The project file is missing or has an incomplete configuration for the <RepositoryUrl /> property. This property specifies the URL of your project's source code repository (typically GitHub, GitLab, or similar). Providing this URL enables users and contributors to access the source code, report issues, and participate in development.

## Impact

- Users cannot access your project's source code
- Contributors cannot submit pull requests or improvements
- Issue tracking and collaboration are not discoverable
- Reduced transparency and trust in the package
- GitHub integration features on NuGet.org are unavailable

## Resolution

### Step 1: Open Your Project File

Locate and open your project file (.csproj, .fsproj, or .vbproj).

### Step 2: Add or Update the RepositoryUrl Property

Ensure the <RepositoryUrl /> property is present in your project's <PropertyGroup> section. Include the .git suffix for consistency:

```xml
<PropertyGroup>
  <RepositoryUrl>https://github.com/yourorg/yourproject.git</RepositoryUrl>
  <!-- Additional properties -->
</PropertyGroup>
```

### Step 3: Validate and Rebuild

Validate your project file for syntax errors and rebuild your project:

```bash
dotnet build
```

## Best Practices

- Use the full repository clone URL (including .git suffix)
- Ensure the repository is public and accessible
- Use HTTPS URLs for security and reliability
- Verify the repository contains the actual source code
- Include the <RepositoryType> property to specify the version control system
- Consider adding issue and pull request URLs via <BugReportUrl /> and <RepositoryUrl />

## Example

```xml
<Project Sdk="Microsoft.NET.Sdk">
  <PropertyGroup>
    <TargetFramework>net8.0</TargetFramework>
    <PackageId>NetEvolve.Defaults</PackageId>
    <Title>NetEvolve Build Defaults and Configuration</Title>
    <RepositoryUrl>https://github.com/dailydevops/defaults.git</RepositoryUrl>
    <RepositoryType>git</RepositoryType>
    <PackageProjectUrl>https://github.com/dailydevops/defaults</PackageProjectUrl>
  </PropertyGroup>
</Project>
```
