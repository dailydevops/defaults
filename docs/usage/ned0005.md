# NED0005: Missing Package Project URL Configuration

## Overview

**Severity**: Warning  
**Category**: Package Metadata  
**Applies To**: Packable Projects

## Description

The project file is missing or has an incomplete configuration for the <PackageProjectUrl /> property. This property specifies the URL to your project's homepage, documentation, or main web presence. Providing this URL helps users find comprehensive information about your package and its ecosystem.

## Impact

- Users cannot find your project's homepage or main documentation
- Reduced engagement and support visibility
- Incomplete package metadata on NuGet.org
- Users may not understand where to get additional help or information

## Resolution

### Step 1: Open Your Project File

Locate and open your project file (.csproj, .fsproj, or .vbproj).

### Step 2: Add or Update the PackageProjectUrl Property

Ensure the <PackageProjectUrl /> property is present in your project's <PropertyGroup> section:

```xml
<PropertyGroup>
  <PackageProjectUrl>https://github.com/yourorg/yourproject</PackageProjectUrl>
  <!-- Additional properties -->
</PropertyGroup>
```

### Step 3: Validate and Rebuild

Validate your project file for syntax errors and rebuild your project:

```bash
dotnet build
```

## Best Practices

- Use a stable, permanent URL that won't change
- Link to the GitHub repository, project website, or documentation portal
- Ensure the URL is publicly accessible
- Verify the URL is correct and functional
- Use HTTPS URLs for security
- Avoid link-shorteners or redirects when possible

## Example

```xml
<Project Sdk="Microsoft.NET.Sdk">
  <PropertyGroup>
    <TargetFramework>net8.0</TargetFramework>
    <PackageId>NetEvolve.Defaults</PackageId>
    <Title>NetEvolve Build Defaults and Configuration</Title>
    <PackageProjectUrl>https://github.com/dailydevops/defaults</PackageProjectUrl>
    <RepositoryUrl>https://github.com/dailydevops/defaults.git</RepositoryUrl>
  </PropertyGroup>
</Project>
```
