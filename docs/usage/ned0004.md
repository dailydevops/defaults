# NED0004: Missing Package Tags Configuration

## Overview

**Severity**: Warning  
**Category**: Package Metadata  
**Applies To**: Packable Projects

## Description

The project file is missing or has an incomplete configuration for the `<PackageTags />` property. Tags are essential for categorizing your NuGet package and improving its discoverability in package repositories. Without relevant tags, users searching for solutions in your domain cannot easily find your package.

## Impact

- Package is not discoverable through tag-based searches on NuGet.org
- Users in your target domain cannot find your package
- Reduced package adoption and visibility
- Poor organization in package search results

## Resolution

### Step 1: Open Your Project File

Locate and open your project file (`.csproj`, `.fsproj`, or `.vbproj`).

### Step 2: Add or Update the PackageTags Property

Ensure the `<PackageTags />` property is present in your project's `<PropertyGroup>` section. Use semicolon-separated tags:

```xml
<PropertyGroup>
  <PackageTags>tag1;tag2;tag3;tag4</PackageTags>
  <!-- Additional properties -->
</PropertyGroup>
```

### Step 3: Validate and Rebuild

Validate your project file for syntax errors and rebuild your project:

```bash
dotnet build
```

## Best Practices

- Include 4-6 relevant, searchable tags
- Use lowercase tags for consistency
- Include domain-specific keywords (e.g., "build", "configuration", "msbuild")
- Include technology stack references (e.g., "dotnet", "csharp", "analyzer")
- Avoid redundant tags already in the package ID or title
- Consider what users would search for when discovering your package
- Use hyphens instead of spaces in multi-word tags

## Example

```xml
<Project Sdk="Microsoft.NET.Sdk">
  <PropertyGroup>
    <TargetFramework>net8.0</TargetFramework>
    <PackageId>NetEvolve.Defaults</PackageId>
    <Title>NetEvolve Build Defaults and Configuration</Title>
    <PackageTags>build;msbuild;configuration;dotnet;csharp;analyzer;standards</PackageTags>
  </PropertyGroup>
</Project>
```
